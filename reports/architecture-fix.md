## Issue 4: except Exception: pass 改善

### 修改總覽

#### clawteam/cli/commands.py（4 處）

| 行號 | 原始 | 修改後 | 原因 |
|------|------|--------|------|
| 1055 | `except Exception:` | `except (json.JSONDecodeError, OSError):` | 讀取 Claude state JSON |
| 1119 | `except Exception:` | `except OSError:` | `os.path.ismount()` 檢查 |
| 1126 | `except Exception:` | `except (OSError, ValueError):` | `TeamManager.discover_teams()` |
| 3031 | `except Exception: pass` | `except (OSError, ValueError) as exc: logging.debug(...)` | workspace cleanup on failure |

#### clawteam/transport/p2p.py（5 處）

| 行號 | 原始 | 修改後 | 原因 |
|------|------|--------|------|
| 123 | `except Exception:` | `except OSError:` | heartbeat peer registration |
| 178 | `except Exception:` | `except (KeyError, TypeError, ValueError):` | peer JSON parsing |
| 213 | `except Exception: pass` | `except (zmq.ZMQError, OSError, ImportError):` | ZMQ send fallback |
| 311-324 | `except Exception: pass` ×3 | `except OSError:` | ZMQ socket/context cleanup |

### 設計原則
- **Cleanup 模式**：`close()` 中嘅 `except OSError` 保留原有語義（best-effort cleanup），但排除咗意外嘅異常類型
- **Fallback 模式**：ZMQ send 失敗改為 catch `zmq.ZMQError`，保留 fallback 到 file transport
- **日誌記錄**：commands.py:3031 嘅 cleanup 失敗改為 `logging.debug()` 而唔係靜默 pass
